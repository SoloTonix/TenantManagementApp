# 1️⃣ High-Level Platform Structure

Your app will now have **three user roles**:

```
Users
   │
   ├── Landlord
   ├── Tenant
   └── Property Agent
```

Each role interacts with the system differently.

---

# 2️⃣ Landlord Side (Property Management)

This side focuses on **managing existing properties and tenants**.

### Core Functions

Landlords should be able to:

• Create properties
• Add units (flats/rooms)
• Invite tenants
• Track rent payments
• Monitor vacancies
• Manage leases

Example structure:

```
Landlord
   │
Property
   │
Units
   │
Tenants
   │
Payments
```

Main goal:

> **Help landlords run rental properties like a business.**

---

# 3️⃣ Tenant Side (Rental Experience)

This side focuses on **rent management and property interaction**.

### Core Functions

Tenants should be able to:

• View rent details
• Pay rent
• View payment history
• Receive reminders
• Request maintenance
• Browse available properties

The last one becomes possible because of the **agent marketplace**.

---

# 4️⃣ Agent Side (Property Marketplace)

This is where your platform becomes **much bigger than a rent manager**.

Agents can:

• List properties
• Upload property images
• Set rent prices
• Add property descriptions
• Manage inquiries from tenants

Example listing:

```
Property: 2 Bedroom Apartment
Location: Lekki
Rent: ₦1,200,000/year
Agent: ABC Realty
```

Tenants can then:

```
View property
Show interest
Contact agent
Schedule inspection
```

This becomes a **mini property marketplace**.

---

# 5️⃣ Marketplace Flow

Typical flow:

```
Agent posts property
        ↓
Tenant browses listings
        ↓
Tenant clicks "Show Interest"
        ↓
Agent receives notification
        ↓
Agent contacts tenant
```

Later you can add:

• inspection booking
• application submission
• tenant screening

# 7️⃣ UX Design Recommendation

To avoid complexity, use **role-based dashboards**.

When users log in:

```
Landlord → Landlord Dashboard
Tenant → Tenant Dashboard
Agent → Agent Dashboard
```

Each dashboard should show only relevant tools.

# 🔟 One Important Product Warning

Be careful not to **mix the two systems too early**.

Keep them separate:

```
Property Marketplace (Agents)
           vs
Tenant Management (Landlords)
```

A property listed by an agent should only become a **managed property** after:

```
Tenant moves in
Landlord starts managing rent
```

This separation keeps the system clean.
